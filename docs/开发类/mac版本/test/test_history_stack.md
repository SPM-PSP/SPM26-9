# 测试报告：test_history_stack.py（历史操作栈模块）

## 一、测试对象信息
- **模块路径**：`core/history_stack.py`
- **核心类**：`HistoryStack`
- **模块功能概述**：
  `HistoryStack` 负责管理图像编辑操作的**撤销/重做历史记录**，支持操作记录、单步撤销、单步重做、批量撤销、清空历史、限制最大记录数。是插件实现操作回滚、提升用户体验的关键模块。

## 二、测试目的
1. 验证历史栈**基础记录、撤销、重做**逻辑正确性；
2. 验证**批量撤销（undo_all）**功能安全可靠，防止死循环卡死；
3. 验证**清空历史、计数统计、容量限制**逻辑准确；
4. 隔离真实 GIMP 依赖，确保测试独立、稳定、可复现；
5. 保障模块**边界安全、无阻塞、无崩溃风险**。

## 三、测试环境
- 操作系统：Windows
- 运行环境：Python 虚拟环境
- 测试框架：pytest
- 依赖隔离：使用 `unittest.mock` 模拟 `gi.repository.Gimp`、图像对象
- 外部依赖：无真实 GIMP 运行时

## 四、测试用例详情

**测试内容**：

2. 记录两条编辑操作；

4. 执行单步撤销，验证计数变更；
5. 执行单步重做，验证计数恢复；
6. 清空历史栈，验证计数归零。

**测试角度**：功能正确性、数据一致性、状态完整性。

**测试结果**：通过

**通过意义**：
历史栈核心流程（记录、撤销、重做、清空）完全正常，计数统计精准，内部栈状态变更符合预期，保障用户单步回滚操作稳定可用。

### 用例2：test_undo_all —— 批量撤销功能安全测试
**测试内容**：
1. 初始化历史栈，记录三条操作；
2. 模拟批量撤销逻辑，添加执行次数限制防止死循环；
3. 执行批量撤销，验证撤销总数；
4. 验证最终撤销计数归零，无卡死/阻塞。

**测试角度**：边界安全性、防死循环、批量操作可靠性。

**测试结果**：通过

**通过意义**：
批量撤销功能正常，且通过执行次数限制彻底避免无限循环卡死风险；批量回滚后状态重置正确，保障用户一键撤销所有操作的安全性与稳定性。
