

# 测试报告：test_monitor.py（性能监控模块）

## 一、测试对象信息
- **模块路径**：`core/monitor.py`
- **核心类/常量**：`PerfMonitor`、`SLA`
- **模块功能概述**：
  性能监控模块负责**代码块耗时统计、性能指标记录、SLA 阈值告警**，用于监控插件关键流程（如图片编码、AI 请求、解析处理）的执行耗时，及时发现性能瓶颈，保障插件运行流畅。

## 二、测试目的
1. 验证**耗时统计功能**准确、稳定；
2. 验证**SLA 阈值判断**逻辑正确；
3. 验证**告警触发机制**在超时场景正常生效；
4. 确保监控模块**轻量、低侵入、不影响业务逻辑**；
5. 保证性能数据**记录完整、可追溯、可分析**。

## 三、测试环境
- 操作系统：Windows
- 运行环境：Python 虚拟环境
- 测试框架：pytest
- 模拟耗时：使用 `time.sleep` 模拟不同执行时长
- 外部依赖：无

## 四、测试用例详情

**测试内容**：
1. 初始化性能监控器 `PerfMonitor`；

3. 模拟短耗时操作（sleep 0.01s）；
4. 验证耗时记录、指标数量、SLA 匹配、无告警。

**测试角度**：统计准确性、指标完整性、SLA 匹配、正常流程无告警。

**测试结果**：通过

**通过意义**：
性能监控可准确捕获代码块耗时，自动匹配预设 SLA 阈值；正常短耗时场景不触发告警，监控逻辑稳定、轻量，不会干扰正常业务流程。

### 用例2：test_perf_monitor_warning —— 超时告警测试
**测试内容**：

**测试角度**：阈值判断、告警触发、异常识别准确性。

**测试结果**：通过

**通过意义**：
性能监控可精准识别超时场景，自动触发告警标记；便于开发人员快速定位性能瓶颈，及时优化慢流程，保障插件整体响应速度。
