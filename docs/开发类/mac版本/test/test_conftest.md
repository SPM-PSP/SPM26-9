
# 测试配置报告：conftest.py（全局测试环境配置）

## 一、配置对象信息
- **文件路径**：`tests/conftest.py`
- **配置类型**：pytest 全局配置与共享 Fixture
- **核心作用**：
  统一管理测试环境、全局 Mock、共享测试对象、路径配置、依赖隔离。为所有测试文件提供**统一、干净、稳定、可复用**的测试基础环境，避免每个测试文件重复配置。

## 二、配置目的
1. **全局依赖隔离**：统一 Mock GIMP 底层依赖，避免真实 GIMP 环境；
2. **共享 Fixture**：提供所有测试共用的模拟对象，减少代码冗余；
3. **路径统一配置**：自动把 `src` 加入 Python 路径，保证模块可导入；
4. **常量模拟**：预定义 GIMP 常用常量，避免测试中因常量缺失报错；
5. **统一测试环境**：保证所有测试用例运行环境一致、结果稳定可复现。

## 三、配置环境
- 操作系统：Windows
- 运行环境：Python 虚拟环境
- 测试框架：pytest
- Mock 工具：`unittest.mock`
- 外部依赖：无真实 GIMP 运行时

## 四、配置内容说明

### 1. 全局 Mock 配置
**配置内容**：
- 自动 Mock `gi.repository.Gimp`、`GLib`；
- 预设常用常量：`ImageBaseType.RGB=0`、`PDBStatusType.SUCCESS=0`、`FillType.TRANSPARENT=0`；
- 拦截真实 GIMP 调用，所有测试运行在纯模拟环境。

**配置意义**：
所有测试**完全脱离真实 GIMP**，不依赖本地安装、不启动 GIMP、不触发 GUI；运行更快、更稳定、跨机器结果一致。

### 2. 共享 Fixture（全局可用）
#### fixture：mock_gimp_image
- 模拟 GIMP 图像对象；
- 提供宽高、图层、撤销组方法；
- 供 engine、layer_manager、history_stack 共用。

#### fixture：mock_drawable
- 模拟 GIMP 图层（Drawable）对象；
- 提供名称、复制、可见性、父图层等方法；
- 供 layer_manager、engine 共用。

#### fixture：mock_pdb_procedure
- 模拟 GIMP PDB 底层接口；
- 提供配置创建、参数设置、运行返回；
- 供 engine 测试参数归一化、动作执行。

**配置意义**：
一次定义、全测试文件自动注入；减少重复代码、统一 Mock 行为、维护更简单。

### 3. 模块路径配置
**配置内容**：
- 将项目 `src` 目录加入 `sys.path`；
- 保证 `core.*` 模块在任意位置都能正常导入。

**配置意义**：
解决“找不到模块”错误，让测试目录结构更规范、运行更省心。

## 五、配置效果与结论
1. **环境隔离彻底**：所有测试不依赖真实 GIMP，运行稳定、无 GUI、无崩溃；
2. **代码高度复用**：共享 Fixture 统一 Mock 对象，减少冗余、降低维护成本；
3. **测试统一规范**：所有测试运行环境一致、Mock 行为一致、结果可复现；
4. **运行效率高**：纯内存 Mock，启动快、执行快、不占用额外资源；
5. **质量保障**：基础环境可靠，避免因环境差异导致测试忽好忽坏。

**最终结论**：`conftest.py` 全局配置完整、合理、稳定，为项目所有测试提供可靠基础环境，保障测试体系稳定、高效、可维护，满足项目交付标准。
