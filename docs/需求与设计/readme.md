## 系统设计文档
- ### 核心层：
   -  系统模块图 ；明确 UI层 怎么调 核心层，核心层 怎么调 AI层。定义好那 3 个文件夹的依赖关系。
   -  调研PDB接口，列出需要使用的核心函数，验证可行性
   -  设计数据流向图，展示“一张图片从加载到修完保存”的全过程。
   -  设计状态机，定义系统有哪些状态（空闲、分析中、修图中、报错），以及状态怎么切换
     
 
- ### AI层
    - 列出AI 能力清单。比如：需要“色彩分析”、“参数生成”两个核心能力。
    - 搭建 AI 开发的本地环境，测试能不能连通 GIMP 的 Python 环境。
    -  设计配置文件格式（.json）。定义好 SRS 里要求的“修图配方”长什么样。
    -  设计时序图
      
 
- ### UI层
    - 梳理界面交互流程。画出草图：点哪个按钮 -> 弹出什么框 -> 显示什么结果。
    - 收集 GIMP 原生界面的风格素材，确定配色方案，确保插件不突兀。
    - 完成界面设计图，切图。
    - 设计用例图
